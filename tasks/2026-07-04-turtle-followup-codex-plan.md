## Confirmed findings (do not re-derive)

1. **0 trade markers — root cause pinned.** Turtle runs are file-backed; the
   execution-markers endpoint falls back to reading `fills.csv`/`trades.csv`
   via `csv.DictReader`, so `ts` arrives as the STRING `"1611878400000"`.
   `_visual_time_values` (`src/okx_quant/api/routes_backtest.py:1796`) sends
   strings to `pd.to_datetime(...)` which returns `NaT` for ms-epoch strings
   → every marker dropped. Reproduced offline against
   `results/ui_turtle_92f5aa71`: `_fallback_execution_markers_from_records`
   returns 0 markers with that run's real fills.csv + trades.csv.
2. **invest_pct slider** (`frontend/view-config.js:1093-1097`) uses
   `onChange` — Preact maps that to the native `change` event which fires
   only on release. Live sync needs `onInput`.
3. **Research risk overrides / Execution profile do NOT conflict with turtle
   at engine level — they are silently ignored.** `_run_turtle_job`
   (`routes_backtest.py:1003`) never reads `req.risk_overrides` /
   `req.execution_profile`. But it's dishonest UX: the form hint
   (`view-config.js:1110`) shows "research override active", and
   `_request_parameters` (`routes_backtest.py:1525`) records both into
   turtle `result.json` as if applied.
4. **Reference CSV provenance** (`new_startegy_海龜/index_parameter_result_full
   (3).csv`, 270 rows): generated by `sweep_params_interactive_full`
   (`new_startegy_海龜/trading_target_func.py:2688`) with
   `own_capital=10000.0, invest_pct=0.25, min_position=0.0001, fee=0.003,
   atr_period=20, single_sys_unit_limit=4, both_sys_unit_limit=4`;
   grid = enter_term_sys1 ∈ 6..30 × leave_term_sys1 ∈ 5..19 with constraint
   `leave_term_sys1 < enter_term_sys1` (25×15−105 = 270 ✓);
   enter_term_sys2=55, leave_term_sys2=20 fixed. Input data was the user's
   local `btc_1H_new.csv` resampled to daily (BTC-USDT-SWAP; exact date
   range unknown — NOT in repo).
5. **3D reference view**: `sweep_params_full.html` = 2×3 grid of five
   `go.Surface` (mdd, win_rate, final_whole_asset, profit_loss_ratio,
   expectancy), Viridis, per-scene z-title, page title carries the fixed
   params (`trading_target_func.py:2855-2885`). Our
   `render_surface_html` (`backtesting/turtle_backtest.py:931`) already
   matches structure; missing only fixed-param title + hovertemplate.

Execution order: C1 → C2 → C3 → C4 → C5 (C1/C2 are independent quick fixes;
C4 is the big one; C5 depends on nothing but is lowest risk last).
