# Experiment Registry

Append-only log of experiments and research runs. Its job is to make results
**reproducible** and to keep an honest **trial count** (R6.3): every search over
parameters or strategies is a trial that inflates the best observed result.

Every experiment must link a hypothesis in [[HYPOTHESIS_LEDGER]]. Do not delete
rows; supersede them. A refuted or disappointing result is as valuable as a
positive one and must stay in the log.

## Family Trial Accounting

Each experiment row records a `family_id`. A family's cumulative `n_trials` is
the sum of grid combinations and retry trials across all rows with the same
`family_id`. CPCV must receive that family cumulative count, not only the grid
count from the current run.

Retry versus new family rule: the same economic mechanism tweaked or bugfixed is
a retry in the same family and counts toward that family budget. A genuinely
different mechanism is a new family with a fresh budget. Relabeling a retry as a
new family to dodge the retry limit or DSR penalty is forbidden.

### Family K-budget (retry-limit accounting)

`K_used` counts a family's **real retry attempts** beyond its original run (a
tweaked / bugfixed / re-cost re-run of the *same* mechanism). Planned,
data-blocked, and stage2-failed rows do **not** consume K. The limit is
`K_limit = 2` (original + 2 retries); a family that reaches it without passing is
shelved and escalated (pipeline stop-condition c). Whether a re-run is a retry
(same mechanism) or a new family is the **checkpoint① #9 human judgment**, not a
mechanical count — the table below is backfilled from the row notes and is
**correctable**. `K_used` is **distinct from** `family_cumulative_n_trials`
(retry *attempts* vs grid *combinations*) and must never be conflated with it.
Machine-readable: rows start with `| F-` (the `| E-` registry parser ignores them).

## Required fields

- **Family ID** groups retry attempts for the same economic mechanism. It is the
  unit used for family-cumulative `n_trials`.
- **Setup** must be enough to reproduce: instrument, date range, config
  reference (path + commit or hash), seed, and data source (DB vs parquet).
- **Trials** is this row's count of parameter/strategy combinations searched.
  Hidden trials are a leakage bug (I13). CPCV receives the family cumulative
  count across rows with the same Family ID. CPCV artifacts must label
  `n_trials_provenance`; a `grid_size_floor` value is a lower bound, not the
  researched trial count.
- **Artifact** points to the reproducible output (a `results/` run_id or file).
  Idealized-fill / in-sample artifacts must be labelled as such (R7.1).
  CPCV evidence must retain raw `path_returns`, or `combined_returns` when path
  assembly is unavailable, plus periods/lengths so DSR can be recomputed offline.
- **Outcome** states the measured result and whether it supported the hypothesis.

## Rules

- Append, never rewrite. To correct an entry, add a new row and note "supersedes
  E-NNN".
- An experiment with no reproducible artifact is anecdote, not evidence.
- Promotion requires walk-forward / CPCV evidence and the
  `docs/ai_collaboration.md` gates, not a single positive row here.
