# 加密貨幣策略合成

本文件把文獻回顧轉換成目前 OKX 系統可驗證的策略假說。這些內容不是實盤建議。每個想法在升級為實盤策略前，都必須通過 OKX 費率、只掛單約束、重播測試或滾動樣本外測試，以及去偏夏普比率檢查。

## 策略一：多層訂單流失衡掛單偏移

- **理論基礎：** Glosten 和 Milgrom，一九八五年；Cont、Kukanov 和 Stoikov，二零一四年；Xu、Gould 和 Howison，二零一八年；Kolm、Turiel 和 Westray，二零二一年。
- **核心邏輯：** 用深度加權的多層訂單流失衡，取代或補強第一檔訂單簿失衡。此訊號用來偏移做市策略中的公允價值，同時所有執行維持只掛單。
- **訊號來源：** OKX 第二層訂單簿更新、最優買賣檔訂單流失衡、深度加權訂單流失衡、短半衰期指數加權平均。
- **部位規則：** 以 `portfolio/sizing.py` 的基礎下單量為起點；當各深度層級訊號分歧過高，或交易流毒性指標超過門檻時降低部位。
- **執行方式：** 計算公允價值等於中間價加上多層訂單流失衡乘以係數，再送入 `strategies/as_market_maker.py` 報價邏輯；只送出 `post_only` 訂單。
- **適用標的：** 優先使用比特幣兌泰達幣永續合約與以太幣兌泰達幣永續合約；其他幣種必須先通過流動性與最小跳動單位檢查。
- **預期優勢：** 來自短週期訂單簿壓力的微結構優勢。
- **風險控制：** 最大庫存、交易流毒性價差倍率、報價穿越時取消、陳舊訂單簿防護、用佇列年齡過濾疑似虛假掛單。
- **與現有系統的銜接：** 擴充 `src/okx_quant/signals/obi_ofi.py`，接入 `strategies/as_market_maker.py` 與 `strategies/obi_market_maker.py`。
- **回測路徑：** 使用 `backtesting/replay.py` 或 `nautilus_backtest.py`，搭配第二層或逐筆資料、掛單成交假設，以及含費用損益。

## 策略二：交易流毒性跳躍風險節流做市

- **理論基礎：** Easley、Lopez de Prado 和 O'Hara，二零一二年；Andersen 和 Bondarenko，二零一四年；近年比特幣交易流毒性與價格跳躍研究。
- **核心邏輯：** 將交易流毒性指標視為毒性與跳躍風險節流器，而不是方向性超額收益訊號。在高毒性狀態下，擴大價差、降低下單量；若庫存已偏危險，可選擇只報單邊。
- **訊號來源：** OKX 成交資料、等成交量分桶、批量成交方向分類、滾動交易流毒性百分位。
- **部位規則：** 正常交易流使用一倍部位；警戒交易流使用零點五倍部位；極端毒性時依庫存狀態降至零到零點二五倍。
- **執行方式：** 報價刷新仍維持只掛單；價差倍率隨交易流毒性百分位與實現波動上升。
- **適用標的：** 成交資料足夠密集的比特幣與以太幣永續合約。
- **預期優勢：** 主要來自避損與降低逆向選擇，而不是獨立方向性收益。
- **風險控制：** 用實現波動與價差過濾降低誤報；避免因單次毒性尖峰直接硬停機。
- **與現有系統的銜接：** 擴充 `src/okx_quant/signals/vpin.py`，將毒性狀態提供給 `risk/risk_guard.py` 與做市策略。
- **回測路徑：** 比較高波動時段中，有無交易流毒性節流的做市損益差異。

## 策略三：加入基差與擁擠度過濾的資金費率套利

- **理論基礎：** Makarov 和 Schoar，二零二零年；加密貨幣永續合約與資金費率文獻；Liu、Tsyvinski 和 Wu 的加密貨幣因子研究。
- **核心邏輯：** 保留現有現貨多頭與永續合約空頭的套利結構，但只在扣除成本後資金費率足夠高，且基差與擁擠度指標未暗示高反轉或爆倉風險時進場。
- **訊號來源：** OKX 歷史資金費率、溢價指數、現貨與永續合約基差、可得時使用未平倉量、實現波動。
- **部位規則：** 依預期資金費率年化、基差標準分數、實現波動與最大爆倉距離決定中性名目本金；再用四分之一凱利或固定比例部位上限約束。
- **執行方式：** 兩腿盡量用掛單建立；只有當方向曝險漂移超過門檻時再平衡。
- **適用標的：** 比特幣與以太幣現貨，加上泰達幣本位永續合約。
- **預期優勢：** 資金費率收益與基差收斂。
- **風險控制：** 資金費率反轉停損、基差擴大停損、爆倉緩衝、交易所風險上限、最大資金配置上限。
- **與現有系統的銜接：** 擴充 `src/okx_quant/strategies/funding_carry.py` 與 `portfolio/sizing.py`。
- **回測路徑：** 使用既有資金費率資料與燭線資料，透過 `backtesting/walk_forward.py` 與壓力時段驗證。

## 策略四：帶有均值回歸半衰期閘門的動態比特幣以太幣配對

- **理論基礎：** Gatev、Goetzmann 和 Rouwenhorst，二零零六年；Bertram，二零零九年；Avellaneda 和 Lee，二零一零年；Tadic 和 Kortchemski，二零二一年。
- **核心邏輯：** 只有當滾動共整合、卡爾曼避險比率穩定性，以及均值回歸半衰期都通過品質閘門時，才交易比特幣與以太幣相對價值。
- **訊號來源：** 比特幣與以太幣永續合約或現貨價格、滾動避險比率、價差標準分數、均值回歸半衰期。
- **部位規則：** 採美元中性或貝塔中性部位；當半衰期上升、價差波動跳升，或避險比率不確定性擴大時降低曝險。
- **執行方式：** 當標準分數絕對值大於二時進場，接近零點五時降倉，接近零或半衰期失效時出場。除非觸發風險停損，否則優先使用掛單。
- **適用標的：** 比特幣兌泰達幣永續合約與以太幣兌泰達幣永續合約；低槓桿版本可使用現貨配對。
- **預期優勢：** 相對價值均值回歸。
- **風險控制：** 共整合破裂停損、標準分數硬停損、最長持有時間、流動性過濾。
- **與現有系統的銜接：** 擴充 `src/okx_quant/strategies/pairs_trading.py`。
- **回測路徑：** 用滾動參數估計做滾動樣本外測試；門檻選擇再加入組合式清洗交叉驗證。

## 策略五：加密貨幣因子殘差籃子

- **理論基礎：** Avellaneda 和 Lee，二零一零年；Liu 和 Tsyvinski，二零一九年；Liu、Tsyvinski 和 Wu，二零二二年；Gu、Kelly 和 Xiu，二零一八年。
- **核心邏輯：** 建立一個高流動性的 OKX 永續合約標的池，先移除比特幣與以太幣等共同市場因子，再用嚴格流動性與成本過濾交易殘差均值回歸或殘差動能。
- **訊號來源：** 高流動性永續合約報酬、比特幣與以太幣因子、成交量、波動、資金費率、殘差標準分數。
- **部位規則：** 對啟用中的殘差訊號使用階層式風險平價或反波動配置；限制單一標的名目本金與總替代幣曝險。
- **執行方式：** 低頻再平衡，優先掛單；若預期優勢低於掛單費用與滑價成本，則不交易。
- **適用標的：** 僅限 OKX 上最具流動性的泰達幣本位永續合約。
- **預期優勢：** 去除共同加密貨幣貝塔後的橫截面殘差錯價。
- **風險控制：** 標的池流動性下限、因子貝塔上限、擁擠交易停損、下架與新聞事件排除。
- **與現有系統的銜接：** 在 `src/okx_quant/strategies/` 下新增策略模組，並擴充 `portfolio/allocation.py`。
- **回測路徑：** 從 `backtesting/vectorbt_scanner.py` 開始，再進行滾動樣本外測試與組合式清洗交叉驗證。

## 策略六：慢速時間序列動能疊加層

- **理論基礎：** Jegadeesh 和 Titman，一九九三年；Moskowitz、Ooi 和 Pedersen，二零一二年；Daniel 和 Moskowitz，二零一六年；Hurst、Ooi 和 Pedersen，二零一八年；Lim、Zohren 和 Roberts，二零二零年。
- **核心邏輯：** 增加一個低換手率的趨勢策略袖套或市場狀態疊加層，只在趨勢強度超過成本，且波動調整後部位可接受時交易。避免純吃單突破式短線動能。
- **訊號來源：** 多週期報酬、實現波動、回撤狀態、資金費率背景。
- **部位規則：** 以目標波動調整曝險，並依回撤降低部位，搭配最大槓桿上限。
- **執行方式：** 回落進場或定期再平衡時優先掛單；只有在實盤政策允許時，風險出場才使用吃單。
- **適用標的：** 比特幣與以太幣永續合約；通過測試後可擴展到高流動性主流幣。
- **預期優勢：** 趨勢延續與危機凸性。
- **風險控制：** 動能崩盤過濾、波動尖峰降風險、最大持有虧損、去偏夏普比率門檻。
- **與現有系統的銜接：** 新增策略模組，重用 `signals/regime.py` 與 `portfolio/sizing.py`。
- **回測路徑：** 用小時線或日線通過 `walk_forward.py` 測試，並納入費用與滑價假設。

## 策略七：基差標準分數均值回歸

- **理論基礎：** Bertram，二零零九年；Makarov 和 Schoar，二零二零年；永續合約資金費率文獻。
- **核心邏輯：** 將永續合約與現貨基差建模為均值回歸價差。當基差標準分數極端，且預期資金費率加收斂收益高於成本時，建立中性部位。
- **訊號來源：** 現貨價格、永續合約標記價格或指數價格、資金費率、基差標準分數、均值回歸半衰期。
- **部位規則：** 用預期收斂優勢除以實現基差波動來縮放部位；再以爆倉距離與資本配置上限約束。
- **執行方式：** 當基差與資金費率偏高時，做多現貨並做空永續合約；只有在借貸、費用與放空約束合理時才考慮反向交易。
- **適用標的：** 比特幣與以太幣現貨和永續合約。
- **預期優勢：** 基差收斂加資金費率收益。
- **風險控制：** 基差擴大停損、最長持有期間、資金費率翻轉出場、最低淨年化門檻。
- **與現有系統的銜接：** 擴充 `funding_carry.py`，或新增 `basis_trading.py`。
- **回測路徑：** 結合既有 `candles_1H.parquet` 與 `funding.parquet`；若缺少基差序列，需新增資料收集。

## 策略八：全策略共用波動狀態過濾器

- **理論基礎：** Moreira 和 Muir，二零一七年；Daniel 和 Moskowitz，二零一六年；交易流毒性文獻；Leland，一九八五年的成本感知避險直覺。
- **核心邏輯：** 使用實現波動、交易流毒性、價差與回撤狀態，作為做市、資金費率套利、配對交易與趨勢策略的共用市場狀態過濾器。
- **訊號來源：** 實現波動、交易流毒性百分位、價差百分位、回撤追蹤器、類似爆倉的價格波動。
- **部位規則：** 全域風險倍率介於零到一之間；各策略自身上限仍然保留。
- **執行方式：** 不直接產生方向性交易。此模組調節報價寬度、下單量、套利配置與配對進場權限。
- **適用標的：** 目前所有 OKX 標的。
- **預期優勢：** 降低回撤與逆向選擇損失。
- **風險控制：** 避免對單一雜訊指標過度反應；至少要求兩個狀態變數共同確認。
- **與現有系統的銜接：** 擴充 `signals/regime.py`、`risk/risk_guard.py` 與 `portfolio/sizing.py`。
- **回測路徑：** 對既有策略進行消融測試，比較加入與不加入狀態倍率的差異。

## 暫不採用的策略

| 想法 | 暫不採用原因 |
|---|---|
| 純吃單日內動能 | 零級費率下，吃單費用會讓小型短週期優勢很難變現。 |
| 超低延遲高頻交易 | 需要機房級或高度最佳化基礎設施，以及可信的佇列模型。 |
| 完全自主強化學習做市 | 在重播模擬器成熟前，模擬到實盤落差與獎勵函數鑽漏洞風險過高。 |
| 社群媒體情緒交易 | 資料品質、機器人操縱與延遲問題，使其不適合作為主要訊號。 |
| 在 OKX 放空比特幣與以太幣選擇權波動 | 流動性、尾部風險與缺少選擇權避險基礎設施。 |
| 廣泛替代幣殘差籃子實盤交易 | 必須先具備穩健標的池過濾、下架處理與成本驗證。 |

## 升級檢查表

| 要求 | 門檻 |
|---|---|
| 資料品質 | 測試期間不得有無法解釋的缺口；時間戳必須對齊。 |
| 成本模型 | 必須納入 OKX 掛單與吃單費率、價差、滑價與未成交。 |
| 驗證方式 | 使用滾動樣本外測試或組合式清洗交叉驗證；樣本內外邊界不得洩漏。 |
| 過度擬合控制 | 去偏夏普比率必須大於或等於零點九五。 |
| 風險 | 最大下單名目本金、回撤停損與熔斷器必須啟用。 |
| 執行 | 預設只掛單；若因風險出場使用吃單，必須明確說明理由。 |
